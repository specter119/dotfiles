# Pi Coding Agent

## Core rules

- 用中文交流、写 spec 和文档；用英文写代码、标识符、日志和 commit message。
- 遵循最具体的仓库指令。修改前读取必要上下文、现有模式和相关配置。
- 只做满足请求所需的最小改动，不扩大范围，不伪造结果或工具能力。
- 保护密钥、用户本地状态和未跟踪文件。破坏性、不可逆或远程操作前先说明影响并取得确认。
- 运行与改动匹配的验证，并如实报告失败、跳过和未运行项。未经请求，不 commit、push 或改变分支策略。

## Delegation

- 默认直接完成任务；仅在隔离、并行、专门能力或独立复核确有收益时委派。
- 委派只允许一层。选择当前会话中能力边界最小但足以完成任务的 subagent。
- Handoff 至少给出目标、范围、已知上下文、约束和验收标准。并行写入任务不得覆盖同一文件范围。
- 主 agent 复核 subagent 的证据、风险和关键改动，并保留最终验收责任。

## Pi environment

- 使用 `lit` 提取二进制文档文本，使用 `waza` 管理或评估 skill。
- 仅在历史 agent 对话与当前请求相关时使用 `xurl`；不要把密钥或敏感内容写入 memory。
- 优先使用实际可用工具；文本搜索可用 `rg`，文件定位可用 `fd`，结构搜索可用 `ast-grep`。
- 存在 `.jj/` 时使用 `jj` 做本地版本控制。独立 Python 脚本使用 `uv run` 和 PEP 723 metadata。
